# In-Network Matrix Multiplication for Large Language Model Inference on Commodity Switches

## Abstract

This work demonstrates that transformer inference can execute natively in network switches by mapping neural network primitives to packet-processing operations. Packet counting, a universal switching primitive, implements matrix multiplication: encoding weights as TCAM firewall rules and activations as packet counts causes the switch's native counting behavior to compute dot products. I validate this approach on commodity Juniper QFX5100 switches (2015) with complete end-to-end inference on GPT-2 (768d, 124M parameters) and GPT-OSS-20B dimensions (2880d, 20B parameters). Architectural innovations including MAC prefix matching (256x TCAM reduction) and batch encoding enable arbitrary model scaling. Performance analysis identifies control-plane access as the bottleneck, not switching fabric, as SSH counter reads consume 92 seconds per layer vs. 2 seconds for packet transmission. This work establishes architectural feasibility for switch-native inference and identifies specific hardware capabilities (stateful registers, data-plane counter access) needed for practical deployment on memory-bound workloads. All code, configurations, and 160+ experiment files and their outputs are available at https://github.com/andrewcampi/in_network_matmul_and_llm_inference
